# Introduction

## What is Dory?

Dory is a **static site generator built for technical documentation**. It turns MDX files and a single `dory.json` config into a fast, beautiful documentation site — with zero boilerplate.

  <Card title="Simple" icon="sparkles" description="Drop in .mdx files, configure one dory.json. No framework knowledge needed." />
  <Card title="Fast" icon="zap" description="Instant hot reload in dev. Quick static builds optimized for production." />
  <Card title="Portable" icon="globe" description="No SSR, no vendor lock-in. Deploy anywhere that serves static files." />
  <Card title="Rich Components" icon="layout-dashboard" description="24 built-in MDX components: code blocks, diagrams, API playgrounds, and more." />
</CardGroup>

## Key Features

<Steps>
  <Step title="Write MDX files">
    Create `.mdx` files in your docs directory. Use standard Markdown with any of Dory's 24 built-in components for rich content.
  </Step>
  <Step title="Configure dory.json">
    Set your site name, colors, fonts, logo, and navigation structure in a single `dory.json` file.
  </Step>
  <Step title="Run the CLI">
    Use `dory build` to generate a static site or `dory preview` to serve a built site on port 3000.
  </Step>
  <Step title="Deploy anywhere">
    The output is plain HTML, CSS, and JS — deploy to Netlify, Vercel, S3, GitHub Pages, or any static host.
  </Step>
</Steps>
