# API Playground Demo

This page demonstrates the new HTTP API playground that allows you to test API endpoints directly from the documentation.

## Basic GET Request

Test a simple GET request with query parameters:

<APIPlayground
  method="GET"
  url="/api/users"
  title="Get Users"
  description="Retrieve a list of users with optional filtering"
  parameters={[
    {
      name: "page",
      type: "number",
      description: "Page number for pagination",
      defaultValue: "1"
    },
    {
      name: "limit",
      type: "number", 
      description: "Number of items per page",
      defaultValue: "10"
    },
    {
      name: "search",
      type: "string",
      description: "Search term to filter users"
    }
  ]}
  baseUrl="https://jsonplaceholder.typicode.com"
/>

## POST Request with Body

Test a POST request with a JSON body:

<APIPlayground
  method="POST"
  url="/api/users"
  title="Create User"
  description="Create a new user account"
  parameters={[]}
  body={{
    "name": "John Doe",
    "email": "john@example.com",
    "username": "johndoe"
  }}
  headers={{
    "Content-Type": "application/json"
  }}
  authType="bearer"
  baseUrl="https://jsonplaceholder.typicode.com"
/>

## Authenticated Endpoint

Test an endpoint that requires authentication:

<APIPlayground
  method="GET"
  url="/api/profile/{userId}"
  title="Get User Profile"
  description="Get detailed profile information for a specific user"
  parameters={[
    {
      name: "userId",
      type: "string",
      required: true,
      description: "The ID of the user to retrieve"
    }
  ]}
  authType="bearer"
  baseUrl="https://jsonplaceholder.typicode.com"
/>

## Features

The API playground provides:

- **Interactive Request Configuration**: Set parameters, headers, request body, and authentication
- **Multiple Auth Types**: Support for Bearer token, API key, and Basic authentication
- **Real-time Response**: See actual HTTP responses with status codes, headers, and formatted JSON
- **Request Timing**: Monitor request duration for performance analysis
- **Error Handling**: Clear error messages for failed requests
- **Copy-Paste Ready**: Generate curl commands and code examples

## Usage in Documentation

To add a playground to your API documentation, use the `APIPlayground` component:

```mdx
<APIPlayground
  method="GET"
  url="/api/endpoint"
  title="Endpoint Title"
  description="Description of what this endpoint does"
  parameters={[
    {
      name: "param1",
      type: "string",
      required: true,
      description: "Parameter description"
    }
  ]}
  authType="bearer"
  baseUrl="https://api.example.com"
/>
```

The playground seamlessly integrates with your existing API documentation and provides a Postman-like experience directly in the docs.
