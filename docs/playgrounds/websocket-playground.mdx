# WebSocket Playground

The `<WebSocketPlayground>` component embeds a fully interactive WebSocket client inside your documentation. Readers can open connections, send messages, and observe responses in real time.

## Live Demo

Connect to a public WebSocket echo server â€” it sends back whatever you send:

<WebSocketPlayground
  url="wss://echo.websocket.org"
  title="Echo Server"
  description="Connect to a public WebSocket echo server. Any message you send will be echoed back immediately."
  servers={[
    { url: "wss://echo.websocket.org", description: "Public echo server" }
  ]}
/>

## Usage in MDX

```mdx
<WebSocketPlayground
  url="wss://api.example.com/ws"
  title="Real-time Events"
  description="Subscribe to live events from the API."
  subprotocols={["events-v1"]}
  headers={{
    "X-Client-Version": "2.0"
  }}
  authType="bearer"
  servers={[
    { url: "wss://api.example.com/ws", description: "Production" },
    { url: "wss://staging.api.example.com/ws", description: "Staging" }
  ]}
/>
```

## Message Types

Messages in the log are color-coded:

| Type | Color | Description |
| --- | --- | --- |
| Sent | Sky blue | Messages you send to the server |
| Received | Green | Messages received from the server |
| Error | Red | Connection errors and protocol errors |
| System | Zinc/gray | Connect, disconnect, and status events |

## Testing JSON APIs

For WebSocket APIs that use JSON messages, send structured messages like:

```json
{ "type": "subscribe", "channel": "updates" }
```

```json
{ "type": "message", "content": "Hello!", "room": "general" }
```

## Props Reference

| Prop | Type | Required | Description |
| --- | --- | --- | --- |
| `url` | `string` | Yes | WebSocket URL (`ws://` or `wss://`). |
| `title` | `string` | No | Display title. |
| `description` | `string` | No | Description shown above the playground. |
| `subprotocols` | `string[]` | No | Pre-configured WebSocket subprotocols. |
| `headers` | `Record<string, string>` | No | Pre-populated handshake headers. |
| `authType` | `'bearer' \| 'apiKey' \| 'basic' \| 'none'` | No | Authentication method. |
| `servers` | `Array<{ url, description, protocol? }>` | No | Server environment options. |
